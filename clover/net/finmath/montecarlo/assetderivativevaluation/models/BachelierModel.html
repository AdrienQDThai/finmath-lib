15    /**
16    * This class implements a (variant of the) Bachelier model, that is,
17    * it provides the drift and volatility specification
18    * and performs the calculation of the numeraire (consistent with the dynamics, i.e. the drift).
19    *
20    * The model is
21    * \[
22    * d(S/N) = \sigma dW, \quad S(0) = S_{0},
23    * \]
24    * \[
25    * dN = r N dt, \quad N(0) = N_{0},
26    * \]
27    *
28    * Note: This implies the dynamic
29    * \[
30    * dS = r S dt + \sigma exp(r t) dW, \quad S(0) = S_{0},
31    * \]
32    * for \( S \). For The model
33    * \[
34    * dS = r S dt + \sigma dW, \quad S(0) = S_{0},
35    * \]
36    * see {@link net.finmath.montecarlo.assetderivativevaluation.models.InhomogenousBachelierModel}.
37    *
38    * The model's implied Bachelier volatility for a given maturity T is
39    * <code>volatility * Math.exp(riskFreeRate * optionMaturity)</code>
40    *
41    * The class provides the model of S to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
42    * \( f = \text{identity} \), \( \mu = \frac{exp(r \Delta t_{i}) - 1}{\Delta t_{i}} S(t_{i}) \), \( \lambda_{1,1} = \sigma \), i.e.,
43    * of the SDE
44    * \[
45    * dX = \mu dt + \lambda_{1,1} dW, \quad X(0) = \log(S_{0}),
46    * \]
47    * with \( S = X \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
48    *
49    * @author Christian Fries
50    * @version 1.0
51    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
52    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
53    */
