13    /**
14    * This class implements a (variant of the) Bachelier model, that is,
15    * it provides the drift and volatility specification
16    * and performs the calculation of the numeraire (consistent with the dynamics, i.e. the drift).
17    *
18    * The model is
19    * \[
20    * d(S/N) = \sigma dW, \quad S(0) = S_{0},
21    * \]
22    * \[
23    * dN = r N dt, \quad N(0) = N_{0},
24    * \]
25    *
26    * Note: This implies the dynamic
27    * \[
28    * dS = r S dt + \sigma exp(r t) dW, \quad S(0) = S_{0},
29    * \]
30    * for \( S \). For The model
31    * \[
32    * dS = r S dt + \sigma dW, \quad S(0) = S_{0},
33    * \]
34    * see {@link net.finmath.montecarlo.assetderivativevaluation.models.InhomogenousBachelierModel}.
35    *
36    * The model's implied Bachelier volatility for a given maturity T is
37    * <code>volatility * Math.exp(riskFreeRate * optionMaturity)</code>
38    *
39    * The class provides the model of S to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
40    * \( f = \text{identity} \), \( \mu = \frac{exp(r \Delta t_{i}) - 1}{\Delta t_{i}} S(t_{i}) \), \( \lambda_{1,1} = \sigma \), i.e.,
41    * of the SDE
42    * \[
43    * dX = \mu dt + \lambda_{1,1} dW, \quad X(0) = \log(S_{0}),
44    * \]
45    * with \( S = X \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
46    *
47    * @author Christian Fries
48    * @version 1.0
49    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
50    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
51    */
